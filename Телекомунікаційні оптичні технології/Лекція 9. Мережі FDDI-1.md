# 🌐 Лекція 9. Мережі FDDI-1

> **FDDI** (Fiber Distributed Data Interface) — набір стандартів для локальних оптичних мереж зі швидкістю **100 Мбіт/с**, розроблений комітетом ANSI X3T9.5 у другій половині 1980-х. Еталон високошвидкісної, відмовостійкої магістральної технології до поширення Gigabit Ethernet.

---

## 📋 9.1 Особливості FDDI
| Параметр | Опис |
|---|---|
| **Швидкість** | 100 Мбіт/с |
| **Середовище** | Оптичне волокно (багатомод/одномод), пізніше — вита пара (UTP Cat.5 / STP) |
| **Топологія** | Фізична: подвійне кільце. Логічна: кільце |
| **Мета розробки** | • Підвищення швидкості до 100 Мбіт/с <br>• Відмовостійкість (автовідновлення) <br>• Ефективна підтримка синхронного та асинхронного трафіку |

### 🔍 Порівняння з Token Ring (IEEE 802.5)
| Характеристика | Token Ring | FDDI |
|---|---|---|
| Випуск маркера | Після повернення свого кадру | Одразу після передачі (`Early Token Release`) |
| Пріоритети / резервування | Використовуються | Відсутні |
| Швидкість | 4 / 16 Мбіт/с | 100 Мбіт/с |
| Масштаб | Сотні метрів | До 200 км шляху маркера, периметр до 100 км |
| Топологія | Одиначне кільце | Подвійне кільце (резерв + моніторинг) |

| Параметр | Значення |
|---|---|
| Метод доступу | Маркерний (детермінований) |

## ⚙️ 9.3 Принцип дії

### 🔁 Подвійне кільце та режим WRAP
- **Первинне кільце (Primary):** Основний шлях передачі даних (рух проти годинникової стрілки).
- **Вторинне кільце (Secondary):** Резерв + безперервний моніторинг цілісності.
- **Режим `WRAP` (згортання):** При обриві або відмові вузла два сусідні пристрої з'єднують первинне та вторинне кільця, відновлюючи єдине логічне кільце. Автоматичне повернення до нормального режиму після усунення несправності.
- **Множинні відмови:** Призводять до мікросегментації (мережа розпадається на ізольовані частини).

### 🎫 Маркерний доступ
1. Циркулює спеціальний службовий кадр — **маркер (token)**.
2. Станція захоплює маркер → передає кадри протягом `THT` (Token Holding Time).
3. **Негайно випускає маркер** у кільце після завершення передачі.
4. Кадр проходить по кільцю, приймаючий вузол копіює його, встановлює прапори (адреса збігається / скопійовано / помилки).
5. **Кадр повертається до джерела**, яке перевіряє прапори та видаляє кадр із кільця.
6. ⚡ **Ефективність:** До **95%** реальної пропускної здатності при навантаженні (порівняно з ~30% у класичному Ethernet).

## 🧩 9.4 Рівні стандарту FDDI
```
[ LLC 802.2 Type 1 ]  → Дейтаграмний режим (без встановлення з'єднань)
        ↓
[   SMT   ] Station Management → Ініціалізація, моніторинг, відновлення, керування станціями
        ↓
[   MAC   ] Media Access Control → Маркер, формати кадрів, CRC-32, розподіл смуги
        ↓
[   PHY   ] Physical → Кодування 4B/5B, тактова частота 125 МГц, паралельно-послідовне перетворення
        ↓
[   PMD   ] Physical Medium Dependent → Тип кабелю, оптичні параметри, роз'єми, обхідні перемикачі
```
🔹 **Розширення PMD (1993–1994):**
- `SMF-PMD` — одномодове волокно + лазерні передавачі.
- `TP-PMD` — мідний кабель: UTP Cat.5 (RJ-45, **CDDI**) або STP Type 1 (DB-9, **SDDI**). Макс. відстань: **100 м**.

## 🔌 9.5 Типи пристроїв та портів

### 🖥️ Класифікація пристроїв
| Тип | Розшифровка | Підключення |
|---|---|---|
| **DAS** | Dual Attachment Station | Безпосередньо до подвійного кільця (порти A + B) |
| **SAS** | Single Attachment Station | Тільки через концентратор (порт S) |
| **DAC** | Dual Attachment Concentrator | До кілець (A+B) + порти M для підключення інших пристроїв |
| **SAC** | Single Attachment Concentrator | Порт S + порти M. Не підключається напряму до кілець |
| **NAC** | Null Attachment Concentrator | Тільки порти M. Використовується для гнучкої комутації |

### 🔘 Типи портів
| Порт | Де використовується | Напрямок сигналу |
|---|---|---|
| **A** | DAS, DAC | Вхід у первинне / Вихід у вторинне (`PI/SO`) |
| **B** | DAS, DAC | Вихід у первинне / Вхід з вторинного (`PO/SI`) |
| **M** | Усі концентратори | Master (`PI/PO`). Підключення DAS, SAS, SAC, інших концентраторів |
| **S** | SAS, SAC | Slave (`PI/PO`). Підключення до порту M концентратора |

> ✅ **Базове правило з'єднання:** `A ↔ B`, `S ↔ M`. Порти A та B з'єднуються між вузлами у зворотному порядку для замикання кілець.

### 🔄 MAC-вузли та особливі режими
- **Single MAC:** 1 вузол на пристрій. Працює лише з первинним кільцем.
- **Dual MAC:** 2 вузли → підтримка повного дуплексу, використання обох кілець для трафіку (**сумарно 200 Мбіт/с**). При активації `WRAP` швидкість падає до 100 Мбіт/с (один вузол простоює).
- **Roving MAC (Блукаючий вузол):** Дозволяє підключати/відключати SAS до порту M без реініціалізації кільця. Може об'єднувати маркери при злитті підмереж. Не жорстко стандартизовано → рідко реалізується у виробників.

## 📌 Ключові висновки
- FDDI — перша масова технологія ЛОМ з гарантованою пропускною здатністю та детермінованим доступом.
- Подвійне кільце + `WRAP` забезпечують високу живучість при одиночних відмовах кабелю або вузла.
- Відмова від пріоритетів та `Early Token Release` значно зменшують затримки та підвищують ефективність до ~95%.
- Згодом витіснена швидшими та дешевшими рішеннями (Fast/Gigabit Ethernet), але залишається навчальним еталоном для проєктування відмовостійких магістралей та кольцевих топологій.
- Сумісність обладнання різних вендорів та стабілізація стандартів зробили FDDI надійним інфраструктурним рішенням 90-х/00-х років.
