# Лекція 9. Мережі FDDI

## 9.1. Особливості FDDI (Fiber Distributed Data Interface)

**FDDI** — набір мережевих стандартів для передачі даних по волоконно-оптичному кабелю зі швидкістю **100 Мбіт/с**.

### Основні характеристики

| Параметр | Значення |
|----------|----------|
| **Швидкість передачі** | 100 Мбіт/с |
| **Тип доступу** | Маркерний (детермінований) |
| **Макс. розмір кадру** | 4500 байт |
| **Макс. кількість станцій** | 500 |
| **Відстань між станціями** | 2 км (багатомод), 20 км (одномод), 100 м (мідь) |
| **Макс. довжина шляху маркера** | 200 км |
| **Периметр подвійного кільця** | 100 км |
| **Середовище передачі** | Оптичне волокно (багатомод/одномод), вита пара (UTP Cat.5, STP) |

### Порівняння з Token Ring (IEEE 802.5)

| Характеристика | Token Ring | FDDI |
|---------------|------------|------|
| **Швидкість** | 4/16 Мбіт/с | **100 Мбіт/с** |
| **Масштаб** | Сотні метрів | **До 200 км** |
| **Відмовостійкість** | Одне кільце | **Подвійне кільце** |
| **Маркер** | Утримується до отримання свого кадру | **Відпускається відразу після передачі** |
| **Пріоритети** | Використовуються | **Не використовуються** |
| **Ефективність** | ~30% при навантаженні | **До 95% при навантаженні** |

### Ключові відмінності в управлінні маркером
1.  **Early Token Release**: станція FDDI випускає маркер безпосередньо після закінчення передачі кадру (не чекає повернення свого кадру).
2.  **Відсутність пріоритетів**: FDDI не використовує поля резервування для виділення ресурсів.

## 9.2. Принцип дії

### Подвійне кільце та режим WRAP
Мережа FDDI будується на основі **двох оптичних кілець** з протилежними напрямками передачі:

```
Нормальний режим:
[Станція] → (Первинне кільце, проти годинникової) → [Станція]
[Станція] ← (Вторинне кільце, резервне, контроль цілісності) ← [Станція]
```

#### Режим відновлення (WRAP)
При обриві кабелю або виході вузла з ладу:
1.  Два пристрої по обидва боки від пошкодження активують вторинне кільце.
2.  Первинне та вторинне кільця об'єднуються в **єдине логічне кільце**.
3.  Мережа повністю відновлює працездатність.
4.  Після усунення несправності — автоматичне повернення до нормального режиму.

> **Множинні відмови**: призводять до мікросегментації мережі (розпаду на незв'язані частини).

### Маркерний доступ до середовища

```
Етапи передачі даних:
1. [Маркер циркулює] → станція без даних миттєво передає його далі
2. [Станція має дані] → захоплює маркер → передає кадри (до часу THT)
3. [Завершення THT] → станція відпускає маркер → наступна станція може передавати
4. [Отримання кадру] → станція перевіряє адресу → копіює (якщо їй) → ретранслює далі
5. [Повернення до джерела] → станція-джерело перевіряє ознаки → видаляє кадр
```

#### Параметри маркерного доступу
*   **THT (Token Holding Time)**: максимальний час утримання маркера станцією для передачі.
*   **TTRT (Target Token Rotation Time)**: номінальний час обігу маркера по кільцю (узгоджується при ініціалізації).
*   **Детермінованість**: гарантований доступ до середовища для кожної станції.

## 9.3. Складові стандарту FDDI

### Рівні протоколу FDDI

```
┌─────────────────────────────┐
│ LLC (IEEE 802.2)            │ ← Логічний контроль каналу (дейтаграмний режим)
├─────────────────────────────┤
│ SMT (Station Management)    │ ← Управління станцією, ініціалізація, аварії
├─────────────────────────────┤
│ MAC (Media Access Control)  │ ← Маркерний доступ, формування кадрів, CRC-32
├─────────────────────────────┤
│ PHY (Physical)              │ ← Кодування 4B/5B, синхронізація, 125 МГц
├─────────────────────────────┤
│ PMD (Physical Medium Dep.)  │ ← Оптичний/мідний інтерфейс, роз'єми, довжини хвиль
└─────────────────────────────┘
```

### Специфікації рівнів

| Рівень | Функції | Стандарти/Параметри |
|--------|---------|-------------------|
| **PMD** | Середовище передачі, оптичні приймачі/передавачі, роз'єми, обхідні перемикачі | Многомод: 1300 нм, 2 км; SMF-PMD (одномод); TP-PMD (мідь, 100 м) |
| **PHY** | Кодування/декодування, синхронізація, керуючі символи | Кодування **4B/5B**, тактова частота **125 МГц**, паралельно→послідовно |
| **MAC** | Управління маркером, адресація кадрів, контроль помилок (CRC-32), розподіл смуги | Формат кадру, правила захоплення/відпускання маркера |
| **SMT** | Ініціалізація кільця, моніторинг з'єднань, обробка аварій, взаємодія рівнів | Протоколи конфігурації, виявлення помилок, відновлення |

#### Розширення рівня PMD (1993-1994)
*   **SMF-PMD**: одномодове волокно, лазерні передавачі, дальність до 20 км.
*   **TP-PMD (CDDI/SDDI)**: вита пара категорії 5 (UTP Cat.5, RJ-45) або екранована пара (STP, DB-9), дальність до 100 м.

## Типи пристроїв та портів FDDI

### Класифікація пристроїв

| Тип | Повна назва | Опис | Порти |
|-----|-------------|------|-------|
| **DAS** | Dual Attachment Station | Станція подвійного підключення безпосередньо до подвійного кільця | A, B |
| **SAS** | Single Attachment Station | Станція одиночного підключення через концентратор | S |
| **DAC** | Dual Attachment Concentrator | Концентратор подвійного підключення до кільця + порти для інших пристроїв | A, B + M-порти |
| **SAC** | Single Attachment Concentrator | Концентратор одиночного підключення (тільки через інший концентратор) | S + M-порти |
| **NAC** | Null Attachment Concentrator | Концентратор без підключення до кільця (тільки локальна комутація) | Тільки M-порти |

### Типи портів

| Порт | Назва | Напрямок сигналу | Призначення |
|------|-------|-----------------|-------------|
| **A** | Primary In / Secondary Out (PI/SO) | Вхід з первинного, вихід у вторинне | Підключення DAS/DAC до кільця |
| **B** | Primary Out / Secondary In (PO/SI) | Вихід у первинне, вхід з вторинного | Підключення DAS/DAC до кільця |
| **M** | Master (Primary In/Out) | Вхід/вихід первинного кільця | Порты концентратора для підключення DAS/DAC/SAS/SAC |
| **S** | Slave (Primary In/Out) | Вхід/вихід первинного кільця | Порты SAS/SAC для підключення до концентратора |

### Правила підключення портів

```
Дозволені з'єднання (V = завжди, N = за умови, P = можливо, X = заборонено):

          | Порт іншого пристрою |
Порт даного| A  | B  | S  | M  |
пристрою   |----|----|----|----|
    A      | V,N| V  | V,N| V,P|
    B      | V  | V,N| V,N| V,P|
    S      | V,N| V,N| V  | V  |
    M      | V  | V  | V  | X  |
```

#### Приклади підключень (Таблиця 9.4-9.5)
*   **DAS ↔ DAS**: A→B, B→A (формування подвійного кільця).
*   **DAC ↔ DAC**: A→B, B→A (розширення магістралі).
*   **SAS → Концентратор**: S→M (підключення через концентратор).
*   **DAS → Концентратор**: A/B → M (Dual Homing для підвищеної надійності).

---

### Конфігурації MAC-вузлів

| Тип | Опис | Переваги/Недоліки |
|-----|------|------------------|
| **Single MAC** | Один MAC-вузол на пристрій | Стандартна конфігурація, вторинне кільце використовується тільки для резерву |
| **Dual MAC** | Два MAC-вузли на пристрій (по одному на кожне кільце) | + Повнодуплексний режим, пропускна здатність до **200 Мбіт/с**<br>− При WRAP один MAC простоює, складніша реалізація |

> **Примітка**: Dual MAC пристрої не набули широкого поширення через складність та неефективне використання при аваріях.

---

### "Блукаючий" MAC-вузол (Roving MAC)

**Проблема**: Підключення нової станції до мережі вимагає ініціалізації кільця (узгодження TTRT), що призводить до тимчасової зупинки передачі даних.

**Рішення**: Концентратор з **roving MAC** (локальним блукаючим вузлом).

```
Механізм роботи:
1. Концентратор має основний MAC + додатковий roving MAC
2. При підключенні нової станції до M-порту:
   - Roving MAC обробляє службові кадри нової станції
   - Основне кільце продовжує працювати без реініціалізації
3. При об'єднанні двох підмереж:
   - Roving MAC затримує один маркер до приходу другого
   - Виконує об'єднання в одне логічне кільце
   - Випускає єдиний маркер
```

> **Обмеження**: Механізм не стандартизований строго, тому має обмежене поширення в обладнанні різних виробників.

## Формат кадру FDDI

```
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│ Pre-   │ SD     │ FC     │ DA     │ SA     │ Info   │ FCS    │ End    │
│ amble  │        │        │        │        │        │ (CRC)  │        │
├────────┼────────┼────────┼────────┼────────┼────────┼────────┼────────┤
│ 8 байт │ 2 байти│ 1 байт │ 6 байт │ 6 байт │ до 4500│ 4 байти│ 2 байти│
│        │        │        │        │        │ байт   │        │        │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
```

*   **Preamble**: синхронізація (8 байт).
*   **SD (Start Delimiter)**: початок кадру.
*   **FC (Frame Control)**: тип кадру (дані/службовий), пріоритет.
*   **DA/SA**: адреси призначення та джерела (6 байт кожна).
*   **Info**: корисне навантаження (до 4500 байт).
*   **FCS**: контрольна сума CRC-32.
*   **End**: кінець кадру.

## Ключові терміни лекції

*   **FDDI**: Fiber Distributed Data Interface — інтерфейс розподілених волоконно-оптичних даних.
*   **Token Passing**: маркерний метод доступу до середовища.
*   **Dual Ring**: подвійне кільце з протилежними напрямками передачі.
*   **WRAP Mode**: режим "згортання" кільця при аварії для відновлення зв'язку.
*   **THT (Token Holding Time)**: час утримання маркера станцією.
*   **TTRT (Target Token Rotation Time)**: цільовий час обігу маркера.
*   **Early Token Release**: раннє відпускання маркера після передачі кадру.
*   **4B/5B Coding**: схема кодування для синхронізації та контролю.
*   **DAS/SAS**: станції подвійного/одиночного підключення.
*   **DAC/SAC/NAC**: концентратори різних типів підключення.
*   **Ports A/B/M/S**: типи портів для формування топології.
*   **Roving MAC**: блукаючий MAC-вузол для динамічного підключення без реініціалізації.
*   **CDDI/SDDI**: FDDI на мідному кабелі (витій парі).
*   **SMT**: Station Management — рівень управління станцією.

---

## Переваги та обмеження FDDI

### ✅ Переваги
*   Висока швидкість (100 Мбіт/с) для свого часу.
*   Детермінований доступ — гарантована смуга пропускання.
*   Висока відмовостійкість завдяки подвійному кільцю.
*   Велика протяжність (до 200 км загального шляху).
*   Ефективність до 95% при високому навантаженні.

### ❌ Обмеження
*   Висока вартість оптичного обладнання.
*   Складність конфігурації та управління.
*   Обмежена підтримка з боку виробників після появи Gigabit Ethernet.
*   Dual MAC конфігурації складні в реалізації та неефективні при аваріях.
