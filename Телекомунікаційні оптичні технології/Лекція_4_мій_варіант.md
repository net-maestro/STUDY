# 📚 Лекція 4: Синхронна цифрова ієрархія (СЦІ / SDH)
> **Стислі конспекти для підготовки** | Інфокомунікаційні мережі

---

## 1. Основи SDH (Synchronous Digital Hierarchy)

### 🔹 Що таке SDH?
**SDH** — система передачі даних, заснована на **синхронізації за часом** передавального та приймаючого пристроїв.

### 🔹 Головна відмінність від PDH

| Параметр | PDH (Plesiochronous) | SDH (Synchronous) |
|----------|---------------------|-------------------|
| **Мультиплексування** | Біт-стаффінг (вставка "пустих" біт) | Синхронне, побайтне чергування |
| **Доступ до каналів** | Покрокове демультиплексування | Прямий доступ до будь-якого каналу |
| **Інтерфейси** | Різні стандарти (Європа/США) | Уніфіковані оптичні/електричні |
| **Управління** | Обмежене | Вбудоване самодіагностування |

### 🔹 Переваги SDH
```
✅ Синхронне мультиплексування (без стаффінгу)
✅ Прямий введення/виведення каналів (Add/Drop)
✅ Стандартні інтерфейси та сумісність ієрархій
✅ Розвинуті засоби управління та моніторингу
✅ Висока надійність (APS, захисне перемикання)
```

## 2. Ієрархія швидкостей SDH

Базовий модуль — **STM-1**. Всі наступні рівні кратні ×4.

| Рівень | Швидкість | Призначення |
|--------|-----------|-------------|
| **STM-1** | 155,52 Мбіт/с | Базовий транспортний модуль |
| **STM-4** | 622,08 Мбіт/с | Агрегація 4×STM-1 |
| **STM-16** | 2,488 Гбіт/с | Магістральні канали |
| **STM-64** | 9,953 Гбіт/с | Ядро мережі |
| **STM-256** | ~40 Гбіт/с | Надвисокі швидкості |

> ⏱ **Важливо:** Кадр передається кожні **125 мкс** (частота 8 кГц) — це забезпечує синхронізацію з голосовими каналами (8 кГц дискретизація).

## 3. Структура кадру STM-1

### 🔹 Матричне представлення
```
┌─────────────────────────────────┐
│ 9 рядків × 270 стовпців = 2430 байт │
└─────────────────────────────────┘
```

### 🔹 Розрахунок швидкості
```
2430 байт × 8 біт × 8000 кадрів/с = 155,52 Мбіт/с ✓
```

### 🔹 Три основні поля кадру

| Поле | Розмір | Призначення |
|------|--------|-------------|
| **SOH** (Section OverHead) | 9×9 = 81 байт | Управління секцією, синхронізація, помилки |
| **AU Pointer** | 1×9 = 9 байт | Вказує початок корисного навантаження |
| **Payload** (VC-4 + POH) | 9×261 = 2349 байт | Корисне навантаження + заголовок тракту |

## 4. Деталізація заголовків (SOH)

### 🔹 RSOH — Regenerator Section OverHead (3×9 байт)
*Обробляється регенераторами*

| Байт | Функція | Опис |
|------|---------|------|
| **A1, A2** | Frame Alignment | Маркери початку кадру (0xF6, 0x28) |
| **C1** | Section Trace | Ідентифікатор секції (SAPI) |
| **B1** | BIP-8 | Контроль помилок попереднього кадру |
| **E1** | Orderwire | Службовий голосовий канал (64 кбіт/с) |
| **F1** | User Channel | Тимчасовий канал для користувача |
| **D1-D3** | DCCR | Канал даних регенератора (192 кбіт/с) |

### 🔹 MSOH — Multiplex Section OverHead (5×9 байт)
*Прозорий для регенераторів, обробляється мультиплексорами*

| Байт | Функція | Опис |
|------|---------|------|
| **B2×3** | BIP-24 | Контроль помилок (крім RSOH) |
| **K1, K2** | APS | Автоматичне захисне перемикання |
| **D4-D12** | DCCM | Канал даних мультиплексної секції (576 кбіт/с) |
| **E2** | Orderwire | Голосовий канал мультиплексної секції |
| **S1** | Sync Status | Статус джерела синхронізації |
| **M1** | REI | Індикація помилок від віддаленого кінця |

> 🚨 **Сигнали тривог:**
> - **MS-AIS** (Alarm Indication Signal) — "всі одиниці", сигнал аварії
> - **MS-RDI** (Remote Defect Indication) — повідомлення про дефект назад передавачу

## 5. Віртуальні контейнери (VC) та інкапсуляція

### 🔹 Принцип «Матрьошки»
```
Триб PDH (E1/E3/E4)
        ↓
   [Контейнер C-n]
        ↓
   + POH (заголовок тракту)
        ↓
[Віртуальний контейнер VC-n] ← Логічний об'єкт маршрутизації
        ↓
   + Pointer (покажчик)
        ↓
[Адміністративний блок AU-n]
        ↓
[Група AUG] → [Модуль STM-N]
```

### 🔹 Типи контейнерів

| Тип | Приклади | Призначення |
|-----|----------|-------------|
| **LO VC** (Low Order) | VC-11, VC-12, VC-2, VC-3 | Низькошвидкісні потоки (2-34 Мбіт/с) |
| **HO VC** (High Order) | VC-3, VC-4 | Високошвидкісні потоки (34-140 Мбіт/с) |

### 🔹 POH — Path OverHead (заголовок тракту, 9 байт для VC-4)
*Супроводжує VC від точки входу до точки виходу*

| Байт | Функція |
|------|---------|
| **J1** | Trace — маркер початку тракту, ідентифікація |
| **B3** | BIP-8 — контроль цілісності вмісту VC |
| **C2** | Signal Label — тип корисного навантаження |
| **G1** | Status — зворотний зв'язок про помилки |
| **F2, F3** | User channel — канал користувача |
| **H4** | Position — індикатор положення у мультифреймі |
| **Z3-Z5** | Reserved — резерв оператора |

## 6. Покажчики (Pointers) — ключова фішка SDH

### 🔹 Навіщо потрібні PTR?
```
❌ Проблема: VC може "плавати" відносно кадру STM через:
   • Різниця тактових частот
   • Фазові зсуви
   • Стаффінг при адаптації

✅ Рішення: Pointer вказує точну адресу початку VC
   • Дозволяє локально виділяти потоки без повного демультиплексування
   • Компенсує відхилення ±3 байти (позитивний/негативний стаффінг)
```

### 🔹 Типи покажчиків
- **AU-PTR** — для високопорядкових контейнерів (VC-3/VC-4)
- **TU-PTR** — для низькопорядкових контейнерів (VC-12/VC-2)

## 7. Схема мультиплексування (спрощено)

```
Потоки PDH:
E1 (2 Мбіт/с) → C-12 → VC-12 → TU-12 → TUG-2
                                      ↓
E3 (34 Мбіт/с) → C-3 → VC-3 → TU-3 → TUG-3 → VC-4 → AU-4 → AUG → STM-1
                                      ↑
E4 (140 Мбіт/с) → C-4 →──────────────┘

TUG-2 ×7 → TUG-3
TUG-3 ×3 → VC-4
```

> 📌 **Правило:** При мультиплексуванні в STM-N перевага надається структурі на базі **AU-4**, а не AU-3 (для сумісності).

---

## 8. Формування STM-N (N=4,16,64...)

### 🔹 Способи агрегації
1. **Каскадне**: 4×STM-1 → STM-4 → 4×STM-4 → STM-16
2. **Пряме**: 16×STM-1 → STM-16 (байтове чергування)

### 🔹 Принцип байтового інтерлівінгу
```
Вхід:  [b0 b0 b0...] [b1 b1 b1...] [b2 b2 b2...]
               ↓
Вихід: [b0 b1 b2 b0 b1 b2...] — чергування байтів
```

> ⚠️ **Умова:** Усі вхідні STM-1 повинні мати однакову структуру корисного навантаження, або потрібне попереднє вирівнювання до TUG/VC рівня.

## 9. Шпаргалка: Ключові терміни

| Акронім | Розшифровка | Суть |
|---------|-------------|------|
| **STM** | Synchronous Transport Module | Транспортний модуль |
| **VC** | Virtual Container | Віртуальний контейнер (маршрутизується) |
| **AU/TU** | Administrative/Tributary Unit | Блок з покажчиком |
| **TUG** | Tributary Unit Group | Група трибних блоків |
| **POH** | Path OverHead | Заголовок кінцевого тракту |
| **SOH** | Section OverHead | Заголовок секції (місцевий) |
| **PTR** | Pointer | Покажчик позиції VC |
| **APS** | Automatic Protection Switching | Захисне перемикання |
| **DCC** | Data Communication Channel | Канал управління мережею |

> 💡 **Порада для запам'ятовування:**  
> Уявіть SDH як "поштову систему":  
> - **Контейнер (C)** = конверт  
> - **POH** = адреса отримувача + трек-номер  
> - **Pointer** = координати конверту у вантажівці (кадрі)  
> - **SOH** = службова інформація для водіїв на ділянці маршруту
