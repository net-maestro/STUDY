# Лекція 6. Структури вищого порядку та вирівнювання швидкостей в SDH

## 6.1. Віртуальний контейнер VC4 (Virtual Container)
Структура вищого порядку, що формується для транспортування потоків великої ємності.

*   **Формування:** Побайтове поєднання **трьох** блоків TUG3.
*   **Додаткові елементи:** 
    *   18 байт баласту.
    *   9 байт трактового заголовка (Path Overhead, **POH**).
*   **Розміри структури:** 
    *   9 рядків × 261 стовпців.
    *   Тривалість кадру: **125 мкс**.
*   **Принцип:** Мультиплексування трьох окремих блоків TUG3 (рис. 6.2).

### Байти трактового заголовка VC4 (POH)
Розподілені у першому стовпці структури VC4 (рис. 6.3).

| Байт | Назва / Функція | Опис |
| :---: | :--- | :--- |
| **J1** | Path Trace | Індикатор тракту (перевірка цілісності з'єднання). |
| **B3** | BIP-8 | Контроль помилок (обчислюється з попередньої передачі VC). |
| **C2** | Signal Label | Сигнальна мітка (індикатор заповнення VC4).<br>`00000000` — не заповнений.<br>`11111111` — заповнений. |
| **G1** | Path Status | Контроль стану тракту (інформація про помилки/аварії для далекого кінця). |
| **F2** | Path User Channel | службовий канал зв'язку. |
| **H4** | Multiframe Indicator | Індикатор надциклу (для структурованих сигналів, напр. VC-12). |
| **F3** | Path User Channel | Додатковий байт для зв'язку (аналог F2). |
| **K3** | APS | Автоматичне резервування (Automatic Protection Switching). |
| **N1** | Network Operator Byte | Моніторинг з'єднань (TCM). |

## 6.2. Адміністративний блок AU4 (Administrative Unit)
Структура, що містить віртуальний контейнер VC4 та покажчик для його локалізації.

*   **Склад:** 
    1.  **Матриця корисного навантаження (PAYLOAD):** 261 стовпець × 9 рядків = **2349 байт** (включає байти VC4).
    2.  **Заголовок блоку (AUOH):** 9 байт, приєднаних до 4-го рядка матриці.
*   **Структура покажчика AU4 (байти H1, H2, H3):**
    *   **H1, H2:** Фіксація значення покажчика.
    *   **H3:** Від'ємне вирівнювання швидкостей.
    *   **Y:** Завантаження числа `1001nn11`.
    *   **U:** Завантаження числа `11111111`.

### Механізм покажчика (Pointer)
*   **Об'єм:** 10 біт у байтах H1 та H2.
*   **Діапазон:** Числа від **0 до 782**.
*   **Одиниця виміру:** Покажчик вказує не на окремі байти, а на **тріади байтів** (групи по 3 байти).
    *   Розрахунок: 2349 байт / 3 = **783 тріади**.
    *   Тріада №0 розташована одразу за байтом H3 (початок 4-го рядка Payload).
*   **Функція:** Вказує початок віртуального контейнера VC4 всередині AU4.

### Прапор нових даних (NDF)
*   **Розташування:** Байти H1 та H2 (4 біти).
*   **Нормальний стан:** `0110`.
*   **Зміна покажчика:** Біти інвертуються у `1001`.
*   **Призначення:** Сигналізує про відхилення початку контейнера VC4 всередині структури AU4.

## 6.3. Синхронний модуль STM-1 (Synchronous Transport Module)
Базовий транспортний модуль SDH.

*   **Склад:** 
    1.  Корисне навантаження (AU4).
    2.  Секційний заголовок (**SOH** - Section Overhead).
*   **Розміри:** 9 рядків × 270 стовпців.
*   **Тривалість:** 125 мкс.

### Секційний заголовок (SOH)
Поділяється на дві частини:

#### 1. RSOH (Regenerator Section Overhead)
*   **Об'єм:** 27 байт (рядки 1-3, стовпці 1-9).
*   **Призначення байтів:**

| Байт | Функція |
| :---: | :--- |
| **A1, A2** | Лінійний синхросигнал (Frame Alignment).<br>A1 = `11110110`, A2 = `00101000`. |
| **B1** | Контрольна сума (помилки) по всьому попередньому кадру STM-1. |
| **J0** | Ідентифікатор структури STM-1 (Section Trace). |
| **D1-D3** | Канали передачі керуючої інформації (DCC) = 192 Кбіт/с (3×64). |
| **E1** | Голосовий службовий канал (Orderwire). |
| **F1** | Канал управління (користувацький). |
| **M** | Спеціальне використання (напр., PontiRadio). |
| **\*** | Резерв для майбутніх стандартів. |
| **X** | Національне використання. |

#### 2. MSOH (Multiplex Section Overhead)
*   **Об'єм:** 45 байт (рядки 5-9, стовпці 1-9).
*   **Призначення байтів:**

| Байт | Функція |
| :---: | :--- |
| **B2** | Контрольна сума (помилки) по попередньому кадру (окрім байтів SOH рядків 1-3). |
| **D4-D12** | Канали передачі керуючої інформації (DCC) = 576 Кбіт/с (9×64). |
| **E2** | Голосовий службовий канал (Orderwire). |
| **K1, K2** | Автоматичний обмін (APS - Automatic Protection Switching). |
| **Z1, Z2** | Канали для майбутніх застосувань. |
| **M1** | Індикація помилок на віддаленому терміналі (REI/FEBE). |
| **S1** | Тип синхронного сигналу STM-1 (Sync Status). |
| **\*** | Резерв для майбутніх стандартів. |
| **X** | Національне використання. |

## 6.4. Вирівнювання швидкостей у системах SDH
Процедура компенсації розсинхронізації між тактовою частотою транспортування (STM-1) та навантаження (VC-4).

*   **Механізм:** Зміщення покажчиків (байтовий стафінг).
*   **Перевага:** Дозволяє вирішити проблеми циклової синхронізації без буферів пам'яті.
*   **Типи вирівнювання:**

### 1. Негативне вирівнювання (Negative Stuffing)
*   **Умова:** Частота STM-1 **нижча**, ніж частота VC-4 (VC-4 "швидший").
*   **Дія:** 
    1.  Інвертуються біти **D** у покажчику (H1/H2).
    2.  3 байти **H3** заповнюються інформаційними бітами (корисними даними).
    3.  У наступному кадрі значення покажчика **зменшується на 1**.
*   **Результат:** VC-4 зміщується у бік зменшення адреси.

### 2. Позитивне вирівнювання (Positive Stuffing)
*   **Умова:** Частота STM-1 **вища**, ніж частота VC-4 (STM-1 "швидший").
*   **Дія:** 
    1.  Інвертуються біти **I** у покажчику (H1/H2).
    2.  3 байти, що слідують за H3, заповнюються **неінформаційними бітами** (стафінг).
    3.  У наступному кадрі значення покажчика **збільшується на 1**.
*   **Результат:** VC-4 зміщується у бік збільшення адреси.

### Правила та обмеження
*   **Достовірність:** Зміна покажчика вважається дійсною, якщо інвертовано мінімум **3 з 5 біт** (I або D).
*   **Захист від помилок:** Після зміни покажчика заборонено змінювати його значення протягом **трьох наступних кадрів**.
*   **Частота:** Максимально допускається 2000 зсувів за секунду (на практиці значно менше).
*   **Стабільність:** Допускається одне зміщення покажчика на 4 цикли.

## Ключові терміни лекції
*   **VC4:** Віртуальний контейнер 4-го рівня (основа для AU4).
*   **AU4:** Адміністративний блок (VC4 + Покажчик).
*   **STM-1:** Синхронний транспортний модуль (базовий кадр SDH).
*   **SOH (RSOH/MSOH):** Секційний заголовок (регенераційний/мультиплексний).
*   **Pointer (Покажчик):** H1, H2 (значення), H3 (можливість стафінгу).
*   **Triad (Тріада):** Одиниця виміру покажчика AU4 (3 байти).
*   **Stuffing (Стафінг):** Вирівнювання швидкостей (Позитивне/Негативне).
*   **NDF:** Прапор нових даних (сигнал зміни покажчика).
*   **APS:** Автоматичне перемикання захисту (байти K1, K2, K3).
