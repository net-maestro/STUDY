# 🌐 Лекція 10. Мережі FDDI-2

> **FDDI-2** розглядає фізичну архітектуру мережі, механізми підвищення надійності (оптичні обхідні перемикачі, Dual Homing), специфікації рівня PMD та стандарти кабельної інфраструктури. Логічна топологія завжди залишається кільцем, фізична може бути гнучко адаптована під задачі інфраструктури.

---

## 📐 10.1 Топології мереж FDDI
Базовий тип з'єднання: **точка-точка** (дозволяє комбінувати різні середовища: одномод/багатомод/мідь на різних ділянках).

| Топологія | Опис | Переваги / Обмеження |
|---|---|---|
| **Точка-точка** | Просте з'єднання двох станцій `SAS` | Найпростіша, обмежена масштабованість |
| **Подвійне кільце** | Кільце з пристроїв `DAS`, кожен вузол є невід'ємною частиною структури | ❌ Відключення/додавання вузла → реініціалізація кільця. Потрібні `OBS` для обходу живлення |
| **Окремий концентратор** | Один `NAC` з внутрішньою шиною, до `M-портів` підключаються `SAS`/`DAS` | Гнучке підключення, можливість `Dual Homing` |
| **Дерево концентраторів** | Ієрархічна зірка з коренем `NAC`, відгалуження до `SAC`/`DAC`/`SAS`/`DAS` | ✅ Зручне для будівель/поверхів, легке адміністрування, ізоляція відмов |
| **Подвійне кільце дерев** | Кореневі `DAC` утворюють магістральне подвійне кільце, до них під'єднані дерева концентраторів | 🌟 Найуніверсальніша. Поєднує відмовостійкість кілець + гнучкість дерев |
| **Dual Homing** | `DAS`/`DAC` підключається портами `A` та `B` до `M-портів` **різних** концентраторів | 🔒 Підвищена надійність. Один канал активний, другий у режимі очікування (автоперемикання при обриві) |

---

## 🔀 10.2 Оптичний обхідний перемикач (OBS)
### 🎯 Призначення
Автоматичне замикання кілець при відключенні живлення `DAS`/`DAC` **без переходу в режим `WRAP`**. Зберігає резерв вторинного кільця та запобігає мікросегментації.

### ⚙️ Принцип роботи
1. Живиться від вузла (`+5 В`).
2. При вимкненні живлення механічно/оптично перемикає сигнал у **байпас-режим** (замикання кілець через себе).
3. **Етапи реконфігурації:**
   - Виключення вузла з кільця
   - Реініціалізація маркерного кільця (`10–150 мс` залежно від масштабу)

### 📏 Обмеження та бюджет втрат
| Параметр | Значення |
|---|---|
| Макс. послідовних OBS | **4 шт.** (через накопичення оптичних втрат) |
| Бюджет лінії (стандарт) | `11 дБ` |
| Втрати на один OBS | `~1,1–2,5 дБ` (залежить від виробника та довжини хвилі) |
| Рекомендований запас | `2–3 дБ` (компенсує старіння волокна та стиків) |

📊 *Приклад розрахунку (тип. конфігурація з 1 OBS, 2 км кабелю, 4 роз'єми):*
| Параметр | Початкові втрати, дБ | Після довгої експлуатації, дБ |
|---|---|---|
| Кабельна система | 3,0 | 3,5 |
| Оптичні з'єднувачі | 1,2 | 2,8 |
| OBS (внесені втрати) | 1,1 | 2,1 |
| **Загальні втрати** | **5,3** | **8,4** |
| **Запас (від 11 дБ)** | **4,7** | **2,6** ✅ |

### 🔧 Технологія виконання
- **Механіко-оптичне перемикання:** поворотні сферичні дзеркала (центро-симетричне відбиття) або рухомі волокна.
- **Одномодові OBS:** складніші у виробництві, вартість у `3–4×` вище. Характеристика: `1280–1650 нм`, втрати `1,3–2,5 дБ`, час спрацьовування `~50 мс`.

## 🔌 10.3 Кабельна система та рівень PMD
Розділення `PHY` / `PMD` дозволяє оновлювати фізичні середовища без змін у кодуванні та тактуванні.

### 📡 Стандарти підрівня PMD
| Середовище | Стандарт | Ключові параметри |
|---|---|---|
| **Багатомодове волокно** | `MMF-PMD` | Градієнтне `62,5/125 мкм` (допускається `50/125`), `λ = 1300 нм`, загасання `≤2,5 дБ/км` |
| **Одномодове волокно** | `SMF-PMD` | Лазерні передавачі, чутливість приймача ідентична `MMF-PMD` (`-31 дБм`) |
| **Вита пара (мідь)** | `TP-PMD` | `UTP Cat.5` (4 пари, `RJ-45`) або `STP Type 1/2` (2 пари, `DB-9`). Макс. відстань `100 м`. Дешевше, простіший монтаж |

⚡ **Оптичні рівні потужності (PMD):**
- Передавач: `-20 ... -14 дБм`
- Приймач: `-31 ... -14 дБм`
- Макс. втрати між станціями: `11 дБ`

### 📶 Функція `Signal Detect` (PMD → PHY)
| Середовище | Механізм контролю |
|---|---|
| **Оптика** | Моніторинг потужності вхідного сигналу. `<br>`🔹 Увімкнення: `>-43,5 дБм`<br>🔹 Вимкнення: `<-45 дБм`<br>🔹 Гістерезис: `1,5 дБ` (запобігає "миготінню" статусу) |
| **Мідь** | Петля постійного струму. Вимірюється цілісність пар за рівнем струму при подачі напруги |

### 🔗 Оптичні з'єднувачі
| Тип | Особливості |
|---|---|
| **MIC** (Media Interface Connector) | 🟢 Стандарт FDDI. Полярна структура, захисний ключ, гарантія правильного підключення `TX/RX` |
| **Duplex SC** | 🌟 Найпоширеніший у світі. Компактніший, дешевший, підходить для `MMF` та `SMF`. Мінус: відсутність фізичного ключа |
| **ST / FC** | Переважно для одномодових ліній (`SMF-PMD`) |

### 🏗️ Змішана кабельна система
Типова архітектура поєднує різні середовища в одній інфраструктурі:
```
🌐 Магістраль (між будівлями) → Одномодове / Багатомодове волокно
    ↓
🏢 Вертикальна (всередині будівлі) → Багатомодове волокно
    ↓
🖥️ Горизонтальна (на поверсі) → Оптичний патч-корд / UTP Cat.5 (TP-PMD)
```
✅ **Універсальність:** одна кабельна траса підтримує `FDDI`, `Fast/Gigabit Ethernet`, `ATM` без заміни інфраструктури.

---

## 📌 Ключові висновки
- Фізична топологія FDDI гнучка: від простого `точка-точка` до складного `подвійного кільця дерев` з ієрархією концентраторів.
- `Dual Homing` та `OBS` забезпечують рівень відмовостійкості, необхідний для фінансових та промислових інфраструктур.
- Рівень `PMD` чітко стандартизує оптичні та мідні інтерфейси, підтримує механізм `Signal Detect` з гістерезисом для стабільності ліній.
- Змішане прокладання кабелів та сумісність коннекторів (`MIC`, `SC`) дозволили FDDI стати еталоном магістральних мереж 1990–2000-х років.
